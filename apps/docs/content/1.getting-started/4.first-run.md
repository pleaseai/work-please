# First Run

After installing and configuring Agent Please, you're ready to run your first agent session.

## Start the Service

```bash [Terminal]
agent-please
```

This starts the orchestrator which will:

1. Parse your `WORKFLOW.md` configuration
2. Connect to your issue tracker
3. Begin polling for eligible issues
4. Open the dashboard at `http://localhost:3000`

## Add a Test Issue

Create a new issue in your GitHub Project (or Asana project) and move it to the state configured in your `WORKFLOW.md`. Agent Please will pick it up on the next poll cycle.

## Monitor the Dashboard

Open `http://localhost:3000` in your browser to see:

- **Running issues** — currently active agent sessions
- **Retry queue** — issues waiting for retry after failure
- **Metrics** — token usage, session duration, turn counts

Click on any issue to see its session detail, including the full agent message history.

## CLI Options

```bash [Terminal]
agent-please [workflowPath] [options]

Arguments:
  workflowPath         Path to WORKFLOW.md (default: ./WORKFLOW.md)

Options:
  --port <number>      Dashboard port
  --verbose            Enable verbose/debug logging
```

## Troubleshooting

### Agent not picking up issues

- Verify your issue is in a state that matches your `WORKFLOW.md` configuration
- Check that your API token has the required permissions
- Look at the terminal logs for error messages

### Dashboard not loading

- Ensure port 3000 is available or specify a different port with `--port`
- Check terminal output for startup errors
