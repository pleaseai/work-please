# Configuration

Agent Please uses a single `WORKFLOW.md` file as its configuration source. This file lives in the root of your target repository and contains YAML front matter for settings and a Liquid template for the agent prompt.

## File Structure

```markdown [WORKFLOW.md]
---
# YAML front matter — configuration
tracker:
  type: github_projects
  api_key: $GITHUB_TOKEN
  project_url: https://github.com/orgs/myorg/projects/1

polling:
  interval_ms: 30000

workspace:
  root: ./workspaces

agent:
  max_concurrent_agents: 3
  max_turns: 10
---

<!-- Liquid template — agent prompt -->
You are working on issue {{ issue.title }}.

{{ issue.body }}
```

## Tracker Configuration

### GitHub Projects v2

```yaml
tracker:
  type: github_projects
  project_url: https://github.com/orgs/myorg/projects/1
  # PAT authentication
  api_key: $GITHUB_TOKEN
  # OR GitHub App authentication
  app_id: $GITHUB_APP_ID
  private_key: $GITHUB_APP_PRIVATE_KEY
  installation_id: $GITHUB_APP_INSTALLATION_ID
```

### Asana

```yaml
tracker:
  type: asana
  api_key: $ASANA_ACCESS_TOKEN
  project_id: '1234567890'
```

## Environment Variable Resolution

Config values matching `$ENV_VAR` are resolved from `process.env` at startup. This keeps secrets out of the file:

```yaml
tracker:
  api_key: $GITHUB_TOKEN # resolved from process.env.GITHUB_TOKEN
```

## Polling Configuration

```yaml
polling:
  interval_ms: 30000 # Poll every 30 seconds
```

## Workspace Configuration

```yaml
workspace:
  root: ./workspaces # Base directory for issue workspaces
```

Each issue gets a dedicated subdirectory under `workspace.root`. The orchestrator validates all workspace paths to prevent path traversal.

## Agent Configuration

```yaml
agent:
  max_concurrent_agents: 3 # Global concurrency limit
  max_turns: 10 # Max turns per agent session
  max_retry_backoff_ms: 300000 # Max retry backoff (5 minutes)
```

### Per-State Concurrency

```yaml
agent:
  max_concurrent_agents: 5
  max_concurrent_agents_by_state:
    In Progress: 3
    Human Review: 2
```

## Workspace Hooks

```yaml
hooks:
  after_create: git clone $REPO_URL .
  before_run: bun install
  after_run: "echo 'Agent completed'"
  before_remove: "echo 'Cleaning up'"
```

## Prompt Template

The body of `WORKFLOW.md` (below the front matter) is a Liquid template rendered with issue context:

| Variable | Description |
|----------|-------------|
| `{{ issue.title }}` | Issue title |
| `{{ issue.body }}` | Issue body/description |
| `{{ issue.number }}` | Issue number |
| `{{ issue.url }}` | Issue URL |
| `{{ issue.state }}` | Current project state/column |

### Conditional Blocks

```markdown
{% if issue.labels contains "bug" %}
This is a bug fix. Focus on reproducing and fixing the issue.
{% else %}
Implement the feature as described.
{% endif %}
```

## Live Reload

Agent Please watches `WORKFLOW.md` for changes. When the file is modified, the config is re-parsed automatically. Invalid configs are rejected and the last known-good config is retained.
