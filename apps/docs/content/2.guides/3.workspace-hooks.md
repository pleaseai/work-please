# Workspace Hooks

Agent Please creates an isolated workspace directory for each issue. You can configure shell hooks that run at different points in the workspace lifecycle.

## Available Hooks

| Hook | When | Use Case |
|------|------|----------|
| `after_create` | After workspace directory is created | Clone repo, install dependencies |
| `before_run` | Before each agent session starts | Pull latest changes, rebuild |
| `after_run` | After each agent session completes | Run tests, clean up artifacts |
| `before_remove` | Before workspace is deleted | Archive logs, push changes |

## Configuration

```yaml [WORKFLOW.md]
hooks:
  after_create: 'git clone $REPO_URL . && bun install'
  before_run: 'git pull origin main && bun install'
  after_run: bun run test
  before_remove: "echo 'Workspace cleanup complete'"
```

## Workspace Isolation

Each issue gets its own directory under `workspace.root`:

```
workspaces/
├── issue-123/          # Workspace for issue #123
├── issue-456/          # Workspace for issue #456
└── issue-789/          # Workspace for issue #789
```

### Git Worktrees

When an issue URL points to a GitHub repository, Agent Please can use git worktrees for isolation:

```
workspaces/
├── issue-123/          # git worktree of the target repo
└── issue-456/          # independent worktree, same repo
```

This allows multiple agents to work on different branches of the same repository simultaneously.

## Path Validation

All workspace paths are validated against `workspace.root` before any agent launch. The path must be a strict child of the root (not equal to it), and symlink traversal is checked. This prevents path traversal attacks.

## Startup Cleanup

On service start, Agent Please automatically cleans up workspaces for issues in terminal states (Done, Won't Do). This keeps the workspace root clean after a service restart.
