# GitHub Projects v2

This guide covers setting up Agent Please with GitHub Projects v2.

## Authentication

Agent Please supports two authentication methods for GitHub:

### Personal Access Token (PAT)

```yaml [WORKFLOW.md]
platforms:
  github:
    kind: github
    api_key: $GITHUB_TOKEN
    owner: myorg

projects:
  - platform: github
    project_number: 1
```

Required PAT scopes:
- `project` — read/write access to projects
- `repo` — read/write access to repositories
- `read:org` — read organization membership

### GitHub App

```yaml [WORKFLOW.md]
platforms:
  github:
    kind: github
    owner: myorg
    app_id: $GITHUB_APP_ID
    private_key: $GITHUB_APP_PRIVATE_KEY
    installation_id: $GITHUB_APP_INSTALLATION_ID

projects:
  - platform: github
    project_number: 1
```

GitHub App auth uses `@octokit/auth-app` to generate installation tokens. When both PAT and App credentials are present, PAT takes precedence.

## Project Setup

1. Create a GitHub Project (Projects v2) in your organization
2. Add status fields matching your workflow states (e.g., "Todo", "In Progress", "Human Review", "Done")
3. Add issues from your repositories to the project

## Status Labels

Agent Please applies labels to track dispatch status:

| Label | Meaning |
|-------|---------|
| `dispatched` | Agent session is running |
| `done` | Agent completed successfully |
| `failed` | Agent session failed |

These labels are created automatically by `agent-please init`.

## Webhook Setup

For real-time updates (instead of polling), configure a GitHub webhook:

1. Go to your repository Settings → Webhooks
2. Set Payload URL to `https://your-server/api/webhooks/github`
3. Set Content type to `application/json`
4. Set a webhook secret and configure it in your environment

```yaml [WORKFLOW.md]
server:
  webhook:
    secret: $GITHUB_WEBHOOK_SECRET
```

## Assignee and Label Filters

Filter which issues the orchestrator picks up:

```yaml [WORKFLOW.md]
projects:
  - platform: github
    project_number: 1
    filter:
      assignee:
        - agent-please-bot
      label:
        - agent-ready
```
