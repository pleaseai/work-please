# Asana

This guide covers setting up Agent Please with Asana as your issue tracker.

## Authentication

Asana uses Personal Access Token authentication:

```yaml [WORKFLOW.md]
platforms:
  asana:
    kind: asana
    api_key: $ASANA_ACCESS_TOKEN

projects:
  - platform: asana
    project_gid: '1234567890'
```

Generate a PAT from your [Asana developer console](https://app.asana.com/0/developer-console).

## Project Setup

1. Create an Asana project for your agent workflow
2. Create sections matching your workflow states (e.g., "To Do", "In Progress", "Done")
3. Add tasks to the project

## Section-Based Status

Asana uses sections as status columns. Tasks are moved between sections to change their state:

```yaml [WORKFLOW.md]
projects:
  - platform: asana
    project_gid: '1234567890'
    active_statuses: [To Do, In Progress]
    watched_statuses: [Human Review]
    terminal_statuses: [Done, Cancelled]
```

**Defaults:** active: `To Do, In Progress` | terminal: `Done, Cancelled`

## Webhook Events

Asana supports webhook events for real-time updates:

```yaml [WORKFLOW.md]
platforms:
  asana:
    kind: asana
    api_key: $ASANA_ACCESS_TOKEN
    webhook_secret: $ASANA_WEBHOOK_SECRET
```

Configure the webhook URL as `https://your-server/api/webhooks/asana` in your Asana app settings.

## MCP Tool Injection

When using Asana, Agent Please injects an `asana_api` MCP tool into each Claude Code session, allowing the agent to make API calls using the service's Bearer token without needing separate credentials.
