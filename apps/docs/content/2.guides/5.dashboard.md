# Dashboard

Agent Please includes a built-in Nuxt-based dashboard for monitoring the orchestrator in real time.

## Accessing the Dashboard

The dashboard is available at `http://localhost:3000` (default) when the service is running:

```bash [Terminal]
agent-please --port 3000
```

## Dashboard Views

### Main Dashboard

The main page shows:

- **Metrics** — active sessions, total token usage, average session duration
- **Running table** — currently active agent sessions with issue title, state, and duration
- **Retry table** — issues in the retry queue with next retry time and attempt count

### Issue Detail

Click any issue to see:

- **Session history** — all agent sessions for this issue
- **Message log** — full agent conversation with tool calls and results
- **Retry history** — failure timestamps and backoff information
- **Token usage** — per-session token consumption

## REST API

The dashboard is backed by a REST API:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/state` | GET | Full orchestrator state snapshot |
| `/api/v1/refresh` | POST | Trigger immediate poll cycle |
| `/api/v1/:identifier` | GET | Per-issue detail |
| `/api/v1/sessions/:sessionId/messages` | GET | Session message history |

## Agent Run History

Session data is stored using libsql (embedded SQLite by default):

- **Embedded mode** — local file at `.agent-please/agent_runs.db`
- **Cloud mode** — Turso remote database for persistent storage

```yaml [WORKFLOW.md]
db:
  turso_url: $TURSO_DATABASE_URL
  turso_auth_token: $TURSO_AUTH_TOKEN
```
